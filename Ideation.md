## 2025.09.11 ver 0.9
- 한국어 특화 의료 멀티모달 AI 벤치마크 개발
- 누구의?  
  : 의료 AI 연구자, 의료데이터 연구기관
- 어떤 문제 해결을 위해?  
  : 의료 AI의 성능 측정을 위한 벤치마크와 데이터셋을 제공하기 위해  
  : 한국 의료 환경을 반영한 멀티모달 데이터셋과 표준화된 벤치마크를 구축하여, 한국어 의료 AI의 성능과 설명가능성을 공정하게 비교할 수 있는 시험대를 제공

- 어떤 기술을 사용해서 만들 것인가?

  | 구분 | 기술 내용 |
  |---|---|
  | 공개 의료 데이터셋 | CheXpert, MedMNIST, MIMIC-CXR, KoMIMIC, KorQuAD |
  | 멀티모달 구성 | 의료 영상(흉부 X-ray, CT 등) + 임상 텍스트(리포트, QA) |
  | 베이스라인 모델 | BioViL, MedCLIP, LLaVA-Med 등 사전학습 의료 멀티모달 모델 |
  | 평가 태스크 | 질병 분류, 이상 소견 판별, 의료 질의응답(VQA), 이미지-텍스트 정합성 평가 |
  | 평가 지표 | Accuracy, AUROC, Robustness, Generalizability 등 |

## 2025.11.11 ver
- 연합 평가(Federated Evaluation) 방식을 활용한 멀티모달 의료 AI 벤치마크 구축
- 병원 간 데이터 직접 공유 없이, 각 기관의 로컬 환경에서 동일한 평가 코드를 실행하여 성능을 비교하는 연합 평가 시나리오를 중심으로 벤치마크 구조 재정의
- 의료 데이터 반출 제한 환경에서도 적용 가능한 평가 프로토콜 및 지표 설계에 초점

## 2025.12.18 ver
- **누구를 위한 프로젝트인가**  
  : 의료 AI 연구자 및 의료 데이터 연구기관을 대상으로, 실세계 의료 데이터 제약 환경에서도 활용 가능한 평가 중심 멀티모달 의료 AI 벤치마크를 구축하고자 함
- **어떤 문제를 다루는가**  
  : 의료 데이터 외부 반출 제한, IRB 승인 부담, 병원 간 데이터 이질성으로 인해 대규모 의료 데이터 확보가 어려운 현실적 문제를 핵심 연구 전제로 설정
- **무엇을 목표로 하는가**  
  : 소규모 의료 데이터 환경에서도 AI 모델의 성능을 공정하게 비교·평가할 수 있는 벤치마크 구조와 평가 방법론을 제시
- **어떤 접근 방식을 사용하는가**  
  : 대규모 의료 데이터로 사전학습된 멀티모달 인코더를 활용하여, 원본 의료 영상과 텍스트를 중간 표현(임베딩 벡터)으로 변환하는 평가 중심 구조를 채택
- **기술적으로 어떻게 해결하는가**  
  : 원본 이미지 증강 대신 임베딩 벡터 공간에서 Manifold Mixup을 적용하여, 소량의 시드 데이터로부터 평가에 필요한 표현 다양성을 확보
- **어떻게 타당성을 검증하는가**  
  : t-SNE 기반 시각화를 통해 원본 데이터와 증강된 데이터의 임베딩 분포를 비교·분석하여, 벡터 기반 증강의 의미적 일관성을 정성적으로 검토
- **왜 이 방식이 의미 있는가**  
  : 사람이 해석할 수 없는 벡터 공간에서의 증강을 통해 개인정보를 보호하면서도, 인공지능 모델이 실제로 활용하는 정보 구조를 효과적으로 확장할 수 있기 때문
- **최종적으로 무엇을 제안하는가**  
  : 개인정보 보호와 연산 효율성을 고려한, 실세계 소규모 의료 데이터 환경에 적합한 멀티모달 의료 AI 평가 벤치마크 설계 방향을 제안
